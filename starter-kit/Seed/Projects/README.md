# Seed/Projects/

Drop in anything that documents work you actually did: case studies, project write-ups, post-mortems,
retro notes, decks, performance-review excerpts. Aim for ~10+.

`seed-project-list` turns these into `Context/project-list.md` — one structured entry per project
(situation, action, result, metrics) that doubles as resume bullets and interview STAR stories. Where a
result or metric is missing, it flags a question for you rather than inventing anything.

(You can delete this README once you've added your files.)
